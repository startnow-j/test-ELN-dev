## 一、模块概述

"我的任务"模块是 BioLab ELN 的**统一任务管理中心**，整合了用户所有与实验记录相关的任务，包括审核管理功能。

**入口**: 侧边栏 → 我的任务

---

## 二、功能分类

### 2.1 Tab 结构

| Tab | 图标 | 显示条件 | 说明 |
|-----|------|----------|------|
| 我的草稿 | FileEdit | 所有用户 | DRAFT状态的实验，我是作者 |
| 待我审核 | CheckCircle | 有审核权限 | PENDING_REVIEW状态，我是审核人 |
| 待我修改 | RefreshCw | 所有用户 | NEEDS_REVISION状态，我是作者 |
| 已锁定 | Lock | 所有用户 | LOCKED状态，我是作者 |
| 解锁申请 | Unlock | 管理员/项目负责人 | 待处理的解锁申请 |

### 2.2 视角切换（管理员专属）

| 视角 | 说明 |
|------|------|
| 普通视角 | 显示我的任务 |
| 全局视角 | 显示所有用户的任务 |

**视角切换按钮**: 页面右上角下拉菜单

## 三、各Tab功能详解

### 3.1 我的草稿

**筛选条件**:
- `reviewStatus === 'DRAFT'`
- `authorId === currentUser.id`（普通视角）
- 无作者限制（全局视角）

**操作**:
| 操作 | 说明 |
|------|------|
| 继续编辑 | 进入实验编辑器 |

**显示信息**:
- 实验标题、状态标签、摘要预览
- 更新时间、关联项目、完整度评分、标签

---

### 3.2 待我审核

**筛选条件**:
- `reviewStatus === 'PENDING_REVIEW'`
- 被指定为审核人 OR 项目负责人（项目内实验）

**操作**:
| 操作 | 说明 |
|------|------|
| 查看 | 查看实验详情 |
| 审核 | 打开审核对话框 |

**审核对话框**:

| 审核选项 | 说明 |
|----------|------|
| 审核通过 | 记录锁定，不可修改 |
| 要求修改 | 退回作者，需填写修改意见 |
| 转交审核 | 转给其他审核人，需选择转交目标 |

**批注附件**:
- 在"要求修改"时可上传批注附件
- 支持 PDF、Word、Excel、图片格式

---

### 3.3 待我修改

**筛选条件**:
- `reviewStatus === 'NEEDS_REVISION'`
- `authorId === currentUser.id`（普通视角）

**操作**:
| 操作 | 说明 |
|------|------|
| 查看反馈 | 查看审核历史和反馈意见 |
| 去修改 | 进入实验编辑器 |

---

### 3.4 已锁定

**筛选条件**:
- `reviewStatus === 'LOCKED'`
- `authorId === currentUser.id`（普通视角）

**操作**:
| 操作 | 说明 |
|------|------|
| 查看 | 查看实验详情（只读） |
| 申请解锁 | 打开解锁申请对话框 |

**解锁申请对话框**:
- 必填：解锁原因
- 提交后进入待处理状态

---

### 3.5 解锁申请

**显示条件**: 管理员或项目负责人

**筛选条件**:
- `status === 'PENDING'`
- 有权限处理的申请

**显示信息**:
- 实验标题、申请人、申请时间
- 解锁原因

**操作**:
| 操作 | 说明 |
|------|------|
| 查看 | 查看实验详情 |
| 批准 | 批准解锁，需填写处理意见 |
| 拒绝 | 拒绝申请，需填写拒绝理由 |

**处理对话框**:
- 显示申请人和解锁原因
- 必填：处理理由

## 四、审核历史组件

### 4.1 界面设计

- **卡片化布局**: 每个操作独立卡片
- **时间指示**: 卡片间向上箭头，最新在下
- **颜色区分**: 不同操作类型不同颜色

### 4.2 显示内容

| 操作类型 | 显示内容 |
|----------|----------|
| 提交审核 | 提交人、提交给谁、提交留言、时间 |
| 审核通过 | 审核人、审核意见、时间 |
| 要求修改 | 审核人、修改意见、批注附件、时间 |
| 转交审核 | 转交人、转交给谁、转交说明、时间 |
| 批准解锁 | 处理人、处理意见、时间 |
| 拒绝解锁 | 处理人、拒绝理由、时间 |

### 4.3 组件位置

`src/components/experiments/ReviewHistory.tsx`

## 五、API接口

### 5.1 实验列表

**端点**: `GET /api/experiments?viewMode=default|global`

**响应**: 实验列表，包含审核请求、审核反馈、附件等

### 5.2 提交审核

**端点**: `POST /api/experiments/[id]/submit`

**请求体**:
```json
{
  "reviewerIds": ["user_id"],
  "note": "提交留言"
}
```

### 5.3 审核操作

**端点**: `POST /api/experiments/[id]/review`

**请求体**:
```json
{
  "action": "APPROVE|REQUEST_REVISION|TRANSFER",
  "feedback": "审核意见",
  "transferToUserId": "user_id",
  "attachmentIds": ["att_id"]
}
```

### 5.4 提交解锁申请

**端点**: `POST /api/experiments/[id]/unlock-request`

**请求体**:
```json
{
  "reason": "解锁原因"
}
```

### 5.5 处理解锁申请

**端点**: `PUT /api/experiments/[id]/unlock-request`

**请求体**:
```json
{
  "requestId": "unlock_request_id",
  "action": "APPROVE|REJECT",
  "response": "处理意见"
}
```

### 5.6 获取待处理解锁申请

**端点**: `GET /api/unlock-requests`

**响应**: 当前用户有权处理的解锁申请列表

## 六、权限控制

### 6.1 视角切换

| 角色 | 普通视角 | 全局视角 |
|------|:--------:|:--------:|
| SUPER_ADMIN | ✅ | ✅ |
| ADMIN | ✅ | ✅ |
| PROJECT_LEAD | ✅ | ❌ |
| MEMBER | ✅ | ❌ |

## 八、用户交互流程

### 8.1 审核流程

```
作者创建草稿 → 关联项目 → 提交审核（指定审核人）
                                    ↓
                              审核人审核
                                    ↓
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
               审核通过          要求修改         转交审核
                    ↓               ↓               ↓
               记录锁定        作者修改重提    其他审核人
```

### 8.2 解锁流程

```
作者申请解锁 → 管理员/项目负责人审批
                        ↓
              ┌─────────┴─────────┐
              ↓                   ↓
           批准解锁            拒绝申请
              ↓                   ↓
          记录解锁           记录保持锁定
              ↓
         作者编辑修改
              ↓
         重新提交审核
```
