# 核心规则速查表

> ⚡ 快速回顾 | 每次关键操作前阅读

---

## 🔑 核心原则

```
改动前评估 → 开发时检查 → 完成后更新 → 保存前总结
```

---

## 📋 检查清单

### 改动前（重大改动）
- [ ] 阅读 SYSTEM_FEATURES_v3.3.md 了解系统架构
- [ ] 阅读相关模块文档
- [ ] 输出影响分析，与用户确认

### 开发时
- [ ] 检查组件/工具函数/Hook是否已存在
- [ ] 复用优先于新建

### 模块完成时（用户确认后）
- [ ] 更新模块文档变更记录
- [ ] 更新 PENDING_TASKS.md
- [ ] 更新相关模块文档
- [ ] 更新 worklog.md

### 保存进度时
- [ ] 工作总结
- [ ] 更新所有相关文档
- [ ] `bun run lint` 检查
- [ ] Git commit + push（需用户提供Token）

---

## 🎯 模块依赖速记

```
用户管理 ──权限──→ 所有模块
项目管理 ←──关联──→ 实验记录 ←──触发──→ 审核管理
                                        ↓
                                   文件管理
                                        ↑
我的任务 ←────────── 聚合 ──────────────┘
```

**改动影响速查**：
| 改动模块 | 必检模块 |
|----------|----------|
| 用户管理 | 所有模块权限逻辑 |
| 项目管理 | 实验记录、审核管理 |
| 实验记录 | 审核、文件、我的任务 |
| 审核管理 | 项目（归档锁定）、我的任务 |

## 🚨 强制规则

1. **数据库变更** → 先备份 db 目录
2. **重大改动** → 先创建 Git 备份标签
3. **完成功能** → 必须测试多角色权限
4. **Token泄露** → 提醒用户立即撤销
