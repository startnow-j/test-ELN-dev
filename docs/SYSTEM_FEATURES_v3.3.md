## 一、系统概述

### 1.1 系统定位

**BioLab ELN** (Electronic Lab Notebook) 是一款面向生物实验室的电子实验记录管理系统，实现实验数据的数字化管理、规范化审核和智能化分析。

### 1.2 目标用户

| 用户类型 | 人数 | 主要场景 |
|----------|------|----------|
| 生物实验室研究人员 | 10-20人 | 日常实验记录、数据管理 |
| 项目负责人/PI | 2-5人 | 项目管理、实验审核 |
| 实验室管理员 | 1-2人 | 系统管理、用户管理 |

### 1.3 核心价值

- **数字化管理**: 替代纸质记录，实现实验数据的电子化存储
- **规范化流程**: 标准化的审核流程，确保数据质量和合规性
- **智能化分析**: AI辅助提取实验关键信息，提升效率
- **可追溯性**: 完整的操作日志和审核历史，支持审计

## 三、核心功能模块

### 3.1 用户管理模块

**入口**: 侧边栏 → 用户管理（仅 SUPER_ADMIN 可见）

**功能**:
- 创建/编辑/禁用用户账户
- 设置用户全局角色（ADMIN/RESEARCHER）
- 查看用户活动状态

### 3.2 项目管理模块

**入口**: 侧边栏 → 项目管理

**功能**:
- 项目创建与编辑
- 成员管理（添加/移除/角色分配）
- 项目文档管理
- 状态流转（ACTIVE → COMPLETED → ARCHIVED）
- 主负责人设置

**视角切换**（管理员专属）:
- 普通视角：显示我创建/我负责/我参与的项目
- 全局视角：显示所有项目

**项目关系标记**:
- 🏠 我创建的
- 👤 我负责的
- 👥 我参与的

### 3.3 实验记录模块

**入口**: 侧边栏 → 实验记录

**功能**:
- 富文本编辑器（TipTap）
- 多附件上传与管理
- AI智能提取（试剂/仪器/参数/步骤）
- 完整度评分系统（满分100分）
- 多项目关联
- 标签管理

**审核流程**:
```
草稿(DRAFT) → 提交审核 → 待审核(PENDING_REVIEW) → 审核通过 → 锁定(LOCKED)
                                           ↓
                                      要求修改
                                           ↓
                                    需修改(NEEDS_REVISION)
```

**视角切换**（管理员专属）:
- 普通视角：显示我的实验记录
- 全局视角：显示所有实验记录

### 3.4 我的任务模块

**入口**: 侧边栏 → 我的任务

**功能分类**:

| Tab | 说明 | 显示内容 |
|-----|------|----------|
| 我的草稿 | 继续编辑 | DRAFT状态，我是作者 |
| 待我审核 | 审核操作 | PENDING_REVIEW状态，我是审核人 |
| 待我修改 | 修改后重提 | NEEDS_REVISION状态，我是作者 |
| 已锁定 | 查看/申请解锁 | LOCKED状态，我是作者 |
| 解锁申请 | 处理解锁 | 仅管理员/项目负责人可见 |

**审核功能**:
- 审核通过：实验记录锁定
- 要求修改：退回作者修改
- 转交审核：转给其他审核人
- 上传批注附件

**解锁申请处理**:
- 批准解锁：记录变为可编辑状态
- 拒绝解锁：记录保持锁定
- 必须填写处理理由

### 3.5 模板管理模块

**入口**: 侧边栏 → 模板管理

**功能**:
- 创建/编辑/删除模板
- 公开/私有模板设置
- 标签管理

## 四、审核流程详解

### 4.1 提交审核

**前置条件**:
1. 实验状态为 DRAFT 或 NEEDS_REVISION
2. 已关联至少一个项目
3. 完整度评分 ≥ 60分

**提交选项**:
- 指定审核人（多选）
- 提交留言

### 4.2 审核操作

**审核人权限**:
- 项目负责人：可审核项目内实验
- 管理员：可审核所有实验
- 不能审核自己的实验

**审核决定**:
| 决定 | 结果 | 后续操作 |
|------|------|----------|
| 审核通过 | LOCKED | 记录锁定，不可修改 |
| 要求修改 | NEEDS_REVISION | 作者修改后重新提交 |
| 转交审核 | PENDING_REVIEW | 转给其他审核人 |

### 4.3 审核历史

**显示内容**:
- 提交审核：提交时间、提交人、提交给谁、提交留言
- 审核通过：审核时间、审核人、审核意见
- 要求修改：审核时间、审核人、修改意见、批注附件
- 转交审核：转交时间、转交人、转交给谁、转交说明
- 解锁批准：处理时间、处理人、处理意见

**界面设计**:
- 卡片化布局
- 向上箭头指示时间顺序（最新的在最下）
- 批注附件可下载

### 4.4 解锁功能

**解锁申请**:
- 作者申请：填写解锁原因
- 申请进入待处理状态

**解锁审批**:
- 审批人：项目负责人、管理员
- 批准：记录变为 NEEDS_REVISION 状态，可编辑
- 拒绝：记录保持锁定

## 五、完整度评分系统

### 5.1 评分规则（满分100分）

| 项目 | 分值 | 规则 |
|------|------|------|
| 标题 | 10分 | 填写即得分 |
| 摘要 | 15分 | ≥20字符满分，否则10分 |
| 结论 | 15分 | ≥20字符满分，否则10分 |
| 关联项目 | 10分 | 关联即得分 |
| 附件 | 30分 | 基础15分 + 每个附件5分（最多+15分） |
| AI提取 | 10分 | 试剂/仪器/参数/步骤各2.5分 |
| 标签 | 10分 | 填写即得分 |

### 5.2 提交要求

- 最低提交分数：60分
- 必须关联项目

## 六、AI智能提取

### 6.1 支持的文件格式

| 格式 | 处理方式 |
|------|----------|
| PDF | pdf-parse 库 |
| DOCX | mammoth 库 |
| DOC | antiword 工具 |
| 图片 | OCR识别 |

### 6.2 提取内容

- **试剂**: 实验使用的化学试剂
- **仪器**: 实验设备仪器
- **参数**: 实验参数设置
- **步骤**: 实验操作步骤

### 6.3 使用方式

1. 上传附件后点击"AI提取"按钮
2. 可选择特定附件进行提取
3. 提取结果可编辑
4. 一键应用到实验记录
